Setup Guide

Step-by-step guides for connecting your data source and setting up your account.

What you'll need

A Google account
A Last.fm account with some listening history
A free Last.fm API key — last.fm/api/account/create

Copy the template sheet

Open the link below and click Make a copy. This saves a personal copy to your own Google Drive — your data stays private and only you can edit it.

Extensions → Apps Script

You'll see the Last.fm sync code already loaded and ready. No changes needed — just close that tab and move on to Step 4.

4

      

        
Step 4

        
Start auto-sync

        
Go back to your sheet. You'll now see a Last.fm menu in the top menu bar, next to Help. Click it and select Start Auto-Update (90 min).

        

          Last.fm
          →
          Start Auto-Update (90 min)
        

        
That's it — your sheet will now automatically pull in new plays from Last.fm every 90 minutes. The 90-minute interval is intentional: some sources like Deezer can have a delay before scrobbles appear on Last.fm, so a longer window helps make sure nothing gets missed.

        

          Seeing a "This app isn't verified" warning? This is completely normal for personal scripts. Google shows this for any script that isn't published to the Play Store. Click Advanced → Go to [app name] (unsafe) → Allow. It is safe — the script only accesses your own sheet.
        

        

          Don't want to wait? You can always pull your latest tracks manually at any time by clicking Last.fm → Fetch Latest Tracks — handy if you just finished a listening session and want your charts to update right away.
        

      

    


    
    

      
5

      

        
Step 5

        
Connect to dankcharts.fm

        
Now link your sheet to the site so your charts load from it. In dankcharts.fm, open Settings (the ⚙ icon), choose Google Sheets, and paste your sheet's URL.

        

          Important: your sheet must be set to "Anyone with the link can view" so the site can read it.

          In your sheet: Share → Anyone with the link → Viewer

Troubleshooting

    

      

        The Last.fm menu doesn't appear in my sheet.

        Close and reopen the sheet. The onOpen function runs when the sheet loads — if you had the sheet open right after making your copy, just refresh the page.
      

      

        No data is being imported.

        Check that your Last.fm username (B1) and API key (B2) in the Settings tab are correct and have no extra spaces. You can test by clicking Last.fm → Fetch Latest Tracks and then going to View → Logs in the Apps Script editor.
      

      

        The site shows "No Google Sheet configured".

        Make sure you've pasted the sheet URL in dankcharts.fm Settings and that the sheet is shared as "Anyone with the link can view".
      

      

        Manual entry gives an error.

        The Web App URL must be redeployed after any script change. In Apps Script: Deploy → Manage deployments → edit the existing deployment → Deploy.

What you'll need

1

      

        
Step 1

        
Create a Last.fm API application

        
Go to last.fm/api/account/create and fill in the form. The values can be anything — here's a quick example:

        

          
FieldWhat to enter

          
Application nameMy Music Charts

          
DescriptionPersonal use

          
Application homepagehttps://dankcharts.fm

          
Callback URLhttps://dankcharts.fm

        

        
After submitting, Last.fm will show you an API Key (32 characters) and a Shared Secret. Copy both — you'll need them in the next step.

        

          Free and instant. No payment or approval needed. The application is only used by you personally.
        

      

    


    
    

      
2

      

        
Step 2

        
Enter your credentials in dankcharts.fm

        
Open Settings (the ⚙ icon), select Last.fm as your data source, then scroll down to the Scrobble Setup section. Expand Use custom API credentials and paste in both values.

        

          Settings ⚙
          →
          Last.fm
          →
          Scrobble Setup
          →
          Use custom API credentials
        

        

          Both fields are required. The Shared Secret is used to sign requests — without it the "Connect" button won't work.
        

      

    


    
    

      
3

      

        
Step 3

        
Connect your Last.fm account

        
Click Connect to Last.fm. Last.fm will open in a new tab asking you to grant access to your account. Click Allow access, then come back to dankcharts.fm and click I've authorized ✓.

        
The status indicator will change to ● Connected as [your username] — you're all set.

        

          This connection is stored in your browser only. If you switch browsers or clear site data, you'll need to reconnect. Each person connects their own Last.fm account — your credentials are never shared.
        

      

    


    
    

      
4

      

        
Step 4

        
Edit and push scrobbles from Raw Data

        
Navigate to the Raw Data tab. Every row has a ✎ edit button on the right. Click it to open the edit panel for that entry — you can fix the artist, track title, album, or timestamp.

        
Once your corrections are ready, click Push to Last.fm. A corrected scrobble will be submitted to your Last.fm history with the details you entered.

        

          Important limitation: Last.fm's public API can add scrobbles but cannot delete or modify existing ones. Pushing a correction adds a new scrobble — it does not replace the original. To fully correct an entry, you'll need to manually delete the original on the Last.fm website: go to your Profile → Scrobbles, find the entry, and delete it there.
        

        

          Tip: Use Apply to all matching entries in the edit panel to fix the same artist/track combination across your entire local history in one go, then push once.
        

      

    


  


  


  
  

    
Troubleshooting

    

      

        The "Push to Last.fm" button doesn't appear in the edit panel.

        You need to complete Step 3 first. Open Settings → Last.fm → Scrobble Setup and check that the status shows "Connected".
      

      

        Clicking "Connect to Last.fm" gives an error.

        Make sure both your API Key and Shared Secret are filled in under "Use custom API credentials". The button won't work with just a username.
      

      

        "Push to Last.fm" shows an error after connecting.

        Your session may have expired. Go to Settings → Disconnect, then reconnect following Step 3 again.
      

      

        The scrobble appears twice on Last.fm.

        That's expected — the push added the corrected version but the original is still there. Delete the original manually on the Last.fm website (Profile → Scrobbles).
